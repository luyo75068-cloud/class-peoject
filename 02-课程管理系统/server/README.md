## 四、核心技术实现思路

### 4.1 路由与布局架构

为实现单页应用（SPA）的流畅体验，同时提升代码复用性和可维护性，采用 React Router 进行路由管理，构建清晰的布局架构：
- 基于 react-router-dom 搭建路由体系，通过 createBrowserRouter 配置路由表，明确划分工作台、课程管理、学生管理、学习总结等页面路由，实现页面无刷新切换，提升用户体验。

- 采用嵌套路由设计，将页面公共部分（左侧菜单、顶部导航）抽离为 MainLayout 公共组件，通过 Outlet 组件实现子路由内容的动态渲染，避免重复编写布局代码，降低维护成本。同时，MainLayout 组件统一管理菜单状态，实现布局与业务页面的解耦。

- 菜单选中状态通过 useState 钩子维护，结合 useLocation 钩子监听路由变化，实现菜单选中状态与当前路由的自动同步，确保用户能清晰感知当前所处页面；同时添加路由守卫思想，避免非法路由访问，提升项目安全性。

### 4.2 前后端数据交互
为保证数据交互的规范性、稳定性和可扩展性，封装统一的请求工具，优化数据请求流程，具体实现思路如下：​

- 基于 Axios 封装全局 request 请求工具，统一配置请求基础路径、请求头、响应拦截器和错误拦截器，避免重复编写请求逻辑，同时便于后续统一维护和扩展（如添加请求缓存、token 认证等）。​

- 结合 useEffect 钩子实现数据请求的时机控制，在页面组件挂载时自动发起 GET 请求获取后端数据，确保页面渲染时数据已准备就绪；同时利用 useEffect 的依赖数组，控制请求的触发时机，避免不必要的重复请求，提升页面性能。​

- 引入 loading 状态管理，在请求发起时设置 loading 为 true，展示加载动画，请求完成（成功或失败）后设置 loading 为 false，关闭加载动画，避免用户因等待时间过长产生困惑，同时防止请求未完成时页面渲染异常，提升交互体验。此外，针对请求失败场景，添加统一的错误提示和降级处理，确保页面不会因接口异常而空白或报错。

## 五、遇到问题与解决方案

### 5.1 前后端数据类型匹配
原因：在写后端接口时没有弄清楚存入的正确的courses_ids的数据类型，多次调试过程中产生了脏数据 ，导致后面多次修改代码也总是无法正确修改课程
解决：一步一步排查问题，查看homework.db数据，删掉重新刷新数据，再次修改代码传递正确数据类型

### 5.2 修改数据没有进行同步更新
原因：没有正确使用Ant Design 机制， 为了实现美化效果在Checkbox.Group 外面套了一个 <div>， 调试修改数据后 数据总是不能正确传递，后面了解到<Form.Item name="course_ids"> 会自动把 value 和 onChange 悄悄传给它的第一个直接子元素。导致Checkbox.Group 拿不到初始值，数据无法进行正确更新。
解决：删掉下面的div盒子 把div样式给到Form.Item，使Checkbox.Group成为Form.Item的第一个直接子元素并且也有正确样式。

### 5.3 Markdown 代码块超长不换行、横向溢出
原因：代码块默认不自动换行，长代码会出现横向滚动条，破坏布局。
解决：在 SyntaxHighlighter 中配置 `whiteSpace: 'pre-wrap'` 和 `wordBreak: 'break-all'`，强制代码自动换行，不再溢出。

### 5.4 TypeScript 复杂类型定义与接口返回不匹配，导致类型报错频发
原因：定义学生、课程相关的复杂类型时，使用了嵌套 interface，且部分字段为可选值和联合类型（如 status 可能为 'active' | 'inactive'），后端接口返回时，部分可选字段缺失、联合类型值不规范，且嵌套对象偶尔为 null，导致 TypeScript 类型校验报错，无法正常渲染数据；同时函数参数类型定义不严谨，导致传递数据时类型不匹配。
解决：优化 interface 定义，给可选字段添加默认值，对联合类型添加类型守卫，判断后端返回值是否符合规范；对嵌套对象添加非空校验（使用 ?. 可选链操作符），避免 null 报错；统一函数参数类型，使用泛型约束接口请求返回数据类型，后端接口返回时严格按照前端定义的类型规范返回，不一致时进行类型转换，消除所有类型报错。

### 5.5 页面路由缓存与组件状态残留，切换页面后数据异常
原因：使用 React Router 切换页面时，没有清除上一个页面的组件状态，导致切换到学习总结页面后，再返回学生管理页面，表单数据、筛选条件仍保留上一次的状态，且部分异步请求的回调函数在页面卸载后仍执行，导致数据渲染错乱、控制台报错；同时路由跳转时没有判断当前页面是否有未提交的表单，直接跳转导致数据丢失。
解决：在组件卸载时（useEffect 清理函数），清除表单数据、筛选条件，取消未完成的异步请求（使用 AbortController）；在路由跳转前添加拦截，判断是否有未提交的表单，提示用户确认后再跳转。
