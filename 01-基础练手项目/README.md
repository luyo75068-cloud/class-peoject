## 三、核心技术实现

1. **主题切换**
   使用 CSS 变量定义全局浅色 / 深色两套配色，将颜色统一维护在根元素 :root 中。
   通过 JavaScript 监听切换按钮，动态给 html 或 body 添加 / 移除 .dark 类，实现全局样式无刷新切换。
   所有模块颜色统一引用变量，保证主题切换一致性。

2. **聊天消息发送**
   点击发送按钮或按下回车 → 获取输入框内容 → 非空校验 → 清空输入框 →
   渲染用户消息 → 展示加载状态 → 发起 API 请求 → 接收返回数据 →
   渲染 AI 消息 → 执行 Markdown 解析 → 代码高亮 → 自动滚动到底部。

3. **API 请求与异步处理**
   使用 fetch + AbortController 实现流式响应，边接收数据边渲染，提升对话流畅度；同时支持暂停生成功能，通过 abort() 中断请求，避免无效资源消耗。

4. **图文对话混合发送**
   支持图片上传预览，使用 FileReader 将图片转为 base64 格式暂存；发送时与文本一起提交给多模态大模型（qwen-vl-plus），实现图文混合对话。

5. **Markdown 解析与代码块自定义渲染**
   基于 marked.js 自定义渲染器，重写 code、heading 标签；自动为代码块添加语言标签和一键复制按钮，并集成 highlight.js 实现语法高亮。


6. **输入框交互与图标状态切换**
   监听输入框 input 事件，根据是否输入内容动态切换图标状态：
   无内容时显示图片、菜单等图标；输入文字后隐藏部分图标，显示发送按钮。
   实现输入状态智能切换，提升体验。

7. **快捷提问卡片功能**
  为首页快捷卡片绑定点击事件，点击后自动填充内容到输入框并直接发送消息，简化操作流程。

8. **Flex 布局与聊天界面排版**
  采用 Flex 弹性布局实现页面结构，消息区域自适应高度；用户消息右对齐，AI 消息左对齐展示，配合头像、气泡样式，还原真实聊天体验。

9. **一键清空对话**
  清空消息列表、中断正在进行的 API 请求、恢复欢迎页面，重置所有交互状态。

## 四、遇到的问题及解决思路

1. **问题**：CSS 盒子模型嵌套混乱，导致按钮消失、位置乱飞、布局错乱
   **原因**：未正确使用 flex 布局，子元素溢出、浮动错误、定位冲突、宽度未限制。
   **解决**：梳理盒子层级，统一使用 flex 布局，添加 overflow:hidden，限制最大宽度，
   调整 justify-content 和 align-items，最终实现稳定布局。

2. **问题**：代码块与语言条之间出现白线、空隙、背景不一致
   **原因**：pre / code 标签自带默认 margin、padding、line-height，与父容器背景不一致。
   **解决**：全局重置间距，强制设置 margin:0; padding:0; background:transparent，统一背景色消除白线。

3. **问题**：输入内容后，发送按钮不显示、图标不切换
   **原因**：updateButtonState 函数未正确绑定输入事件.
   **解决**：监听 input 事件实时判断内容，动态添加 / 移除 has-content 类控制图标显示。

4. **问题**：AI 回复时无法暂停、重复请求冲突
   **原因**：未使用 AbortController 管理请求，多个流式请求同时执行导致混乱。
   **解决**：创建全局控制器，发送新请求前自动中断上一个请求，暂停按钮手动中断。

5. **问题**：消息发送后页面不自动滚动到底部
   **原因**：DOM 渲染未完成就执行滚动，导致高度获取错误。
   **解决**：每次更新消息后立即执行 scrollTop = scrollHeight，保证滚动到最新消息。
